Moai 2.0ドキュメントのご案内

ここではMoai Ver2.0とそれに付随するシステムすべてに関するドキュメントが掲載されています.
上のリンクは階層構造となっており、現在あなたは「Moai2.0ドキュメント」の「入り口」を閲覧されている状態です.

「Moai2.0ドキュメント」には多くの記事があり、上のリンクの２段目に列挙されているものたちがそれに該当します.
ここではそれらの概要を紹介しましょう.
  • インストールガイド
    Moai Ver2.0をインストールしてそれを初めて起動するまでを解説してあります.
    インストールといっても、Moai公式サイトより配布されている実行バイナリのzipファイルをダウンロードし、
    それを展開してmoaiという実行ファイルを実行するだけの単純なものです.
    しかし、Moaiはマルチプラットフォームであるため、それらはWindows, Linux, Androidなど環境によって異なります.
    よってそれぞれの環境について別途解説してあります.

  • Moai本体
    Moai Ver2.0 のエンジン本体に関して説明したリファレンスです.
    主にMoaiエンジンの設定画面に関するものや、フィルタ機能、MoaiCGIおよびその認証、新防御システムXhrDMZ、
    不要な接続のブロック機能(広告のカットにも応用できます)、その他の応用的な機能についてです.
    若干難易度が高い内容であり、またこれを知らなくてもさしあたって困らないと思われるため、
    初心者の方はEasterの方をまず読むことをお勧めします.

  • Easter
    Easterは仮想環境を利用してサイトを閲覧するためのクライアントビューアであり、
    これはこのツールの全機能を解説したリファレンスとなります.
    EasterによってユーザのMoai利用は旧バージョンに比べて飛躍的に簡単になりました.
    (Easterを使えば、旧バージョンのMoaiでは必須であったローカルプロキシの設定をする必要がありません).
    ほとんどの方はこのEasterの「仮想ブラウジング」機能をメインで使う形になると思います.
    Easterにはさらに画像分類機能等がありますが、これを使わない方は「コレクション」の項目は気にする必要はありません.

  • VirtualUSERS
    VirtualUSERSは仮想環境を初心者でもボタン一発で簡単に構築するためのツールであり、
    これはこのツールの使い方を解説したものです.
    ほとんどの方はこのVirtualUSERSをMoaiをインストール直後最初に一回実行し、
    以降は必要に応じて、主にルータリセット等でIPアドレスを変更した直後に実行する形になると思います.

  • CustomBoy
    CustomBoyとは、Moai/Easterの仮想環境を詳細にカスタマイズするエキスパートユーザ向けのツールです.
    (従って初心者の方がこのツールを使う必要はまずないと思われます)
    我々が提供する仮想化ツールの中でもこれは最強のカスタマイズ性を有するものです.
    ここではこのツールの使い方のガイドラインを示し、またUIのおける各項目の意味について辞書的なリファレンスを掲載しています.
    （このツールの開発と各種情報の提供は高速ヤキソバ変換氏にご協力頂きました. 彼女にはこの場を借りてお礼申し上げます)

  • HowToコンパイル
    Moaiを公式サイトより配布されている実行バイナリではなく、
    githubにあるソースコードからビルドして使いたいといった潔癖な方、あるいは技術派な方もおられると思います.
    これはそんな方向けのドキュメントです.
    Moaiの全ソースコード(ライブラリも含む)をコンパイルする方法について、割とゼロベースから解説してあります.
    コンパイラ(MinGWを使用)のインストール方法から解説している徹底ぶりですので、
    コンパイルなんてしたことない方でも根気よく読めば出来ると思います(多分)
    またVCプログラマの方向けにVCでコンパイルする方法、CygwinやMSYS、LinuxやAndroid向けに
    コンパイルする方法も記載してあります.

  • Moai CGI Developers
    Moai CGI は Moaiとは独立した単独のCGIスクリプトを実行します.
    従って、あなたがCGIスクリプトを独自に開発し、それをMoai CGI上で実行させることもできるのです！
    ここではこのようなWebアプリケーションを独自に作ってみたい野心溢れる方向けに、その開発方法を解説します.
    そして最終課題としてProxyFinderを例として挙げ、このようなツールが開発できるレベルを目指します.
