    参考

    Moaiの初回起動時、Windowsのファイアウォール機能により、以下のダイアログが表示されることがあります.
    外部マシンからのアクセスを許すサーバ機能を持つアプリケーションを起動した場合、
    Windowsファイアウォールはこの確認ダイアログを出します.
    技術的に言えば、プログラムのソースコード上において、
    INADDR_ANYを指定してWinSock2 APIでのbind を実行した場合となります.





    この場合「ブロックしない」あるいは「アクセスを許可」等を選んでおくとよいでしょう.
    また仮に「ブロックする」や「キャンセル」を選んでも、このツールをローカルプロキシとして使う分には
    特に問題ありません.

    Moaiを他のLAN上のマシンなどから経由して利用をしたいというような極めて特殊な用途の場合は、
    このブロックを解除する必要があるかもしれません.

    終了させたい場合はMoaiのウィンドウを閉じましょう.

Linux版の入手およびインストール方法
    インストールといってもzipをダウンロードして単に展開するだけですが、
    以下に手順としてこれを説明しておきます.

  • 1. Moai Ver1.1.8はzipファイルとなっており、以下より入手できます.

    https://github.com/mr-moai-2016/znk_project/releases/v1.1.8

    Assetsにある bin_for_linux.zip がLinux版となります.
    これをダウンロードしましょう.

  • 2. ダウンロードしたzipファイルを展開します.

  • 3. 展開後のディレクトリの中へ移動します.
    以下のように入力しEnterを押します.

    • cd bin_for_linux

  • 4. 展開後ディレクトリの中に moaiファイルがあることをlsコマンドで確認してください
    (この moai ファイルが実行バイナリ本体となります).
    万一にこれに実行権限が付与されていない場合は以下のコマンドを入力します.
    • chmod 755 moai
    ./moai

    インストールについては以上です.
    これでいつでもmoaiを起動できる状態になりました.

    Moaiを明示的に終了させる必要性は通常ありませんが、
    終了させたい場合はCtrl+Cでプロセスをkillして構いません.

    アンインストールするには展開フォルダをそのまま削除する形になります.

Cygwin版の入手およびインストール方法
    Moai Ver1.1.8ではCygwin版のバイナリは提供しておりません.
    そもそもWindows版がある中、わざわざCygwin版を選ぶといった方は
    ソースコードから自分でビルドして使いたいという方がほとんどだと思うので.

    ソースコードからのビルドについてはHowToコンパイルをご覧ください.

    Moaiが無事起動できるようになった後は、
    Moaiを明示的に終了させる必要性は通常ありませんが、
    終了させたい場合はCtrl+Cでプロセスをkillして構いません.
    アンインストールするには展開フォルダをそのまま削除する形になります.

Android版の入手およびインストール方法
    Android版のインストールは、いわゆる一般的なAndroidアプリとはその方法が異なっています.
    Google Play Storeからインストールするだとかそういったオシャレなものは、
    Googleが好き勝手に変更する仕様に追従すんの超面倒なので残念ながらサポートしておりません.
    お手数ですが、基本的にはLinuxに準じた方法で、インストールおよび起動する形になります.
    Linuxに準じた方法といっても、さほど難しくはないと思います.
    以下に手順としてこれを説明しておきます.

  • 1. Moai Ver1.1.8はzipファイルとなっており、以下より入手できます.

    https://github.com/mr-moai-2016/znk_project/releases/v1.1.8

    Assetsにある bin_for_android がAndroid版となります.
    これをダウンロードしてください.
    通常のARMベースCPUであれば動作すると思われます.
    それ以外にも armeabi_v7a や android-x86 版バイナリなどが含まれています.
    前者は新しいCPUに最適化されたもので場合によっては高速に動作しますが、
    古い端末(ARMv5、v6 端末など)と互換性がありません.
    また後者は、PC(x86)上のAndroidエミュレータなどで動作させるためのバイナリとなります.
    通常のユーザはこれらを実行する必要はありません.

  • 2. Androidのホーム画面に以下のような「端末エミュレータ」アイコンがあります.
    これより端末エミュレータウィンドウを開きましょう.


  • 3. ホームディレクトリへ移動します.
    以下のように入力しEnterを押します.

    • cd

    最初なのでこの様子を示したスクリーンショットも一緒に載せておきます.
    (以降の操作の分については省略します).

  • 4. 上記1でダウンロードした zipファイルは /storage/sdcard0/Download ディレクトリなどに保存されている場合があります.
    (システム構成によっては違う場所に保存されているかもしれません. その場合はその場所に読み替えてください).
    ホームディレクトリ以外でMoaiを実行すると、書き込み許可等の関係で何かと不便なことが起こりがちですので、
    まずはダウンロードした zip ファイルをホームディレクトリへ移動またはコピーする必要があります.
    以下のように入力しEnterを押します.

    • cp /storage/sdcard0/Download/bin_for_android.zip ~/

  • 5. zipファイルを展開します.
    以下のように入力しEnterを押します.
    (以降の説明でも同様に入力後にEnterを押してコマンドを実行してください).

    • unzip bin_for_android.zip

  • 6. 展開後のディレクトリの中へ移動します.

    • cd bin_for_android

  • 7. moai_for_android.sh を実行します.
    sh ./moai_for_android.sh


    • インストールについては以上です.
    これでいつでもmoaiを起動できる状態になりました.
    上記 1 から 7 までの手順は最初の一回だけとなり、
    これらを一度済ませてしまえば以降は上記 2、3、6、7 だけでMoaiが起動可能です.

    moai_for_android.sh において特に明示的にCPUのアーキテクチャを指定したい場合は
    引数にそのアーキテクチャのABI名を指定します.
    例えばandroid-x86 エミュレータ上で動作させたい場合./moai_for_android.sh x86 と実行します.

    Moaiを明示的に終了させる必要性は通常ありませんが、
    終了させたい場合はプロセスごとkillして構いません.
    言っている意味がわからない方は「端末エミュレータウィンドウ」を閉じましょう.

    アンインストールするには展開フォルダをそのまま削除する形になります.
