VirtualUSERS：ボタン一つで仮想環境を構築する
    VirtualUSERSを使うために必要なことは、以下の二つだけです.
    (既にMoaiを起動できる状態であることを前提とします.)

  • 1. moai(Windowsならmoai.exe)を起動する.

  • 2. ブラウザから http://localhost:8124 を開く.
    • これでうまくいかない場合は http://127.0.0.1:8124 で試してみましょう.

    お使いのブラウザに「Welcome to Moai+Easter World」のページが表示されたでしょうか？
    表示されたなら、そこからVirtualUSERSをクリックし、VirtualUSERSの画面を表示させます.
    あとはその画面で「VirtualUSERS」ボタンを押すだけで仮想環境の構築が完了です.

    特に新しくMoaiをダウンロードした直後、初めてのMoai起動時はまずVirtualUSERSで、
    仮想環境をあなた用に構築しておいた方よいでしょう.

    Note

    この構築は別に何回行っても構いません.
    あなたが納得するブラウザ環境となるまで繰り返し構築し直すこともできます.

    Moaiの旧バージョン1.1.8からバージョンアップされた方へ

    Moaiの旧バージョンMoai Ver1.1.8ではMoaiのConfig画面から「VirtualUSERS Initiate」ボタンを押して仮想環境の構築を行っていました.
    Moai Ver2.0ではこれは廃止され、VirtualUSERSはMoai CGIアプリケーションの一つになっています.
    この点ご注意ください.

CustomBoyの概要
    通常、CustomBoyの思考ルーチンは与えられた環境カテゴリを入力として
    それに矛盾しない範囲において最適なUA/Post変数値をランダムに自動的に構築します.
    このランダマイズはCustomBoyの持つ知識ベースを基に確率的アルゴリズムによって行われます.
    さらにCustomFeatureでは、その際に参照するマシン/ブラウザ固有の各要素値を明示的に上書き指定し
    生成されるFingerprint値の究極なるカスタマイズを行うができます.
    CustomBoyは以下の３つの機能より構成されます.

  • CustomUSERS
  • CustomCookie
  • CustomFeature
    • 正確に言えばVirtualUSERSもCustomBoyの一機能として実装されておりますが、ここでは別枠扱いします.

    以下ではこれら３つについて使い方を解説します.

    目次に戻る

CustomUSERS
    CustomUSERSの使用方法として次のステップをご紹介します.
    尚、この機能はエキスパート向けです.
    1. Step1ボタンにより仮想環境を構築します.
      このときCustomUSERSにおいてほぼすべての情報の構築は自動的に完了しており、
      一連の情報は選択された環境カテゴリに矛盾しないよう適切にセットされています.

    2. このままの状態でも全くかまいませんが、さらに各固有の値に拘りのある方は
      これを微調整して修正した上でUA/Post変数の更新ボタンを押します.
      これによりその調整が反映されたUA/Post変数がセットされます.

    3. Step2ボタンを押すことで最終状態を確定させましょう.
      これで完了です.

    目次に戻る

CustomFeature
    CustomFeatureの使用方法として次のステップをご紹介します.
    尚、この機能はエキスパート向けです.
    1. Step1ボタンにより仮想環境を構築します.
      このときCustomFeatureにおいてこの構築結果は自動的に反映されており、
      一連の値はこの結果に矛盾しないよう自動的に文字列がセットされています.

    2. このままの状態でも全くかまいませんが、さらに各固有の値に拘りのある方は
      画面の下の方にある「UA/POST変数生成のためのFingerprint設定」と
      「UA/POST変数生成のためのSnippet(Javascriptによる判定コード)設定」の各値を修正します.
      これらの値の意味は、各値のリンクをクリックしていだだければヒントページが表示されますので参考にしてください.
      (このヒントは高速ヤキソバ変換氏によって作成して頂いたものです).
      これを微調整して修正した上でUA/Post変数の更新ボタンを押します.
      これによりその調整が反映されたUA/Post変数がセットされます.

    3. Step2ボタンを押すことで最終状態を確定させましょう.
      これで完了です.
